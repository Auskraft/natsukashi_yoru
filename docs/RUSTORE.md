## Юридические документы

- Внутри приложения: экран согласия при первом запуске + раздел «Документы»
  (Пользовательское соглашение, Политика конфиденциальности, Обработка ПДн).
- Для поля «Политика конфиденциальности (URL)» в RuStore: опубликовать
  [`../legal/index.html`](../legal/index.html) (например, GitHub Pages: Settings →
  Pages → ветка `main`, папка `/legal`) и указать полученный адрес.

## Что подготовить (ассеты стора)

- [ ] **Иконка** 512×512 PNG (есть исходник `assets/icon/icon.png`).
- [ ] **Скриншоты телефона** — минимум 3–4 (лобби + пара игр). Сделать на
      устройстве/эмуляторе.
- [ ] (Опц.) Графическая шапка/баннер, если потребует витрина.

## Сборка релиза

1. Настроить подпись релиза (keystore). ⚠️ Ключи НЕ коммитить — они уже в
   `.gitignore` (`*.keystore`, `key.properties`).
2. Собрать:
   ```bash
   flutter build appbundle --release   # AAB (предпочтительно)
   # или
   flutter build apk --release         # APK
   ```
3. Загрузить артефакт в консоль RuStore, заполнить карточку (описание, категория,
   возраст, скриншоты, URL политики), отправить на модерацию.

## Чек-лист перед отправкой

- [ ] applicationId = `com.auskraft.natsukashi_yoru`, версия указана.
- [ ] Иконка и сплэш на месте (`dart run flutter_launcher_icons` /
      `flutter_native_splash:create` уже применены).
- [ ] `flutter analyze` чисто, `flutter test` зелёные.
- [ ] Релиз подписан, AAB/APK собран.
- [ ] Описание, скриншоты, возраст 0+, URL политики заполнены.
- [ ] Документы доступны в приложении (экран согласия + раздел «Документы»).

## Заметки

- Сеть используется только для подгрузки шрифтов (Google Fonts) при первом
  запуске. Если для витрины важна полная автономность — можно забандлить шрифты
  локально (отдельная задача) и убрать упоминание сети из политики.
- Реклама, встроенные покупки, аккаунты и сбор персональных данных отсутствуют.
